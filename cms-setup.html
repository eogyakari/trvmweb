⚙️ Content Management

Netlify CMS — Setup Guide

Follow these steps to connect your site to a live Content Management System where you can write devotions, manage books, organise media, and edit site content.

🚀 Open CMS Dashboard 📊 Browser Admin
⚠️ Before accessing /admin/ — complete the setup below first. The CMS requires your site to be hosted on Netlify and connected to a GitHub repository. The browser-based admin (admin.html) works immediately without setup.

Setup Steps

📦 Step 1 — Push Site to GitHub

1
Create a GitHub accountGo to github.com and sign up if you don't have an account.
2
Create a new repositoryClick New → name it trvm-website → set to Private → click Create.
3
Upload all site filesIn your new repo, click Add file → Upload files and upload all HTML files, the admin/ folder, _content/ folder, netlify.toml, and any images.

🌐 Step 2 — Connect to Netlify

1
Create a Netlify accountGo to netlify.com and sign up with your GitHub account.
2
Add new siteClick Add new site → Import an existing project → GitHub. Select the trvm-website repo.
3
Configure build settingsLeave Build command blank. Set Publish directory to . (a single dot). Click Deploy site.
4
Set a custom domain (optional)In Site settings → Domain management, add your custom domain (e.g. trvm.org).

🔐 Step 3 — Enable Netlify Identity & Git Gateway

1
Enable Netlify IdentityIn Netlify: Site Settings → Identity → Enable Identity.
2
Set registration to Invite OnlyUnder Identity → Registration, choose Invite only so only you can log in.
3
Enable Git GatewayUnder Identity → Services → Enable Git Gateway. This allows the CMS to commit changes to GitHub.
4
Invite yourselfUnder Identity → Invite users, enter your email. Check your inbox and accept the invite to set your password.

🚀 Step 4 — Access the CMS

Once setup is complete, visit:

https://your-site.netlify.app/admin/

Log in with the email and password you set in the invite. You'll see the full CMS dashboard with access to all content collections.

How It Works

⚡ The Content Flow

  1. You write a devotion or upload a book in the CMS dashboard at /admin/
  2. Netlify CMS commits the content as a Markdown or JSON file to your GitHub repository
  3. Netlify detects the new commit and redeploys your site (takes ~30 seconds)
  4. The new content is live on your website

All content is stored as plain text files in your GitHub repo — no database needed, fully version-controlled, and you own all your data.

📁 Files Created by This Setup

  • admin/index.html — The CMS dashboard interface
  • admin/config.yml — All collections and field definitions
  • netlify.toml — Netlify deployment configuration
  • _content/ — Folder where all CMS content is stored
  • _content/settings/general.json — Site-wide settings
  • _content/devotions/*.md — Devotion articles
  • _content/books/*.md — Book entries
  • _content/videos/*.md — Video entries
  • _content/gallery/*.md — Gallery albums
